# ДЗ1 команды XRUST_BEZE 

### Пост
#### 1. Создание
- Вводимый текст отображается без искажений
- При нажатии на "Enter" происходит перевод строки в тексте
- Клик вне поля ввода останавливает ввод текста
- Если в поле ввода не введен текст, отображается плейсхолдер "Что нового?"
- Выбранное вложение прикрепляется к посту
- Можно добавить несколько вложений поочередно
- Можно выбрать и добавить несколько вложений сразу
- При нажатии на крестик вложение удаляется
-  <b>BUG: вложение сначала сдвинуто вниз на 3 строки, а потом прижимается к тексту и равномерно смещается вниз</b>

#### 3. Удаление
- При нажатии на иконку мусорки появляется окно "Удалить пост?"
- При нажатии на "Удалить" в окне "Удалить пост?" публикация удаляется и сразу же исчезает из ленты
- При нажатии на "Отмена" в окне "Удалить пост?" публикация остается в ленте
- Нельзя удалить пост, автором которого не являешься

#### 4. Взаимодействие с постом
- При нажатии на иконку лайка она перекрашивается в красный цвет и счетчик лайков увеличивается на 1
- При повторном нажатии на иконку лайка лайк убирается, иконка становится серой и счетчик уменьшается на 1
- При нажатии на иконку комментариев под постом происходит переход на страницу с комментариями
- Если комментариев к посту нет, отображается надпись "Вы будете первым, кто оставит комментарий"
- Если в поле ввода комментария не введен текст, отображается плейсхолдер "Оставить комментарий"
- Нельзя опубликовать {пустой комментарий / комментарий, содержащий только пробелы}
- При вводе комментария нельзя перейти на новую строку, весь текст пишется в одной строке
- При нажатии на "Enter" комментарий публикуется

### Лента
- Ленту можно просматривать в 3х режимах: "Новости", "Сообщества", "Друзья"
- В ленте отображаются все посты за все время, соответствующие текущему режиму
- Посты сортируются сначала по дате изменения, затем по дате публикации
- Новые посты подгружаются автоматически при прокрутке вниз
- В режиме "Новости" показываются все посты
- В режиме "Сообщества" показываются только публикации сообществ, на которые подписан пользователь
- В режиме "Друзья" показываются только публикации друзей пользователя
- При отсутствии постов (или после последнего доступного поста) отображается сообщение "Больше постов нет"

### Друзья
- Раздел "Друзья" имеет 3 вкладки: "Друзья" (взаимная подписка), "Подписчики", "Подписки"
- При отсутствии друзей/подписчиков/подписок, отображаются сообщения "У вас нет друзей"/"У вас нет подписчиков"/"У вас нет подписок"
- В каждом из разделов напротив имени юзера отображается иконка сообщения, при нажатии на которую происходит переход в чат с этим юзером
- При нажатии на аватар юзера происходит переход на его страницу
- "Друзья": напротив имени юзера отображается кнопка "Удалить из друзей"
- "Друзья": При нажатии на кнопку "Удалить друга" для конкретного пользователя появляется окно "Удалить друга?"
- "Друзья": При нажатии на "Удалить" в окне "Удалить друга?" подписка на пользователя удаляется и он перемещается во вкладку "Подписчики"
- "Друзья": При нажатии на "Отмена" в окне "Удалить друга?" друг остается в списке друзей
- "Подписчики": напротив имени юзера отображается кнопка "Добавить в друзья"
- "Подписчики": При нажатии на кнопку "Добавить в друзья" для конкретного пользователя пользователь сразу же перемещается во вкладку "Друзья"
- "Подписки": напротив имени юзера отображается кнопка "Отписаться"
- "Подписки": При нажатии на кнопку "Отписаться" для конкретного пользователя пользователь сразу же удаляется из списка подписок
- Если в строку поиска не введен текст, отображается плейсхолдер "Найти друга"
- Поиск происходит автоматически, нет необходимости нажимать кнопку "Найти"
- При очищении поисковой строки происходит возвращение к списку друзей/подписчиков/подписок
- Поиск работает одинаково во всех трех вкладках: показывает список совпадений среди всех зарегистрированных пользователей без каких-либо указаний на особый статус пользователя (друг/подписчик/подписка)
- Найденные пользователи отображаются в виде списка, на каждого пользователя можно перейти, дополнительных кнопок и действий и не предусмотрено
